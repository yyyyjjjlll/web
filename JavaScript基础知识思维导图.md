## 🌐 第一模块：JavaScript 基础知识

### 📌 一、JavaScript 介绍

### 1.1 JavaScript 是什么？

```
JavaScript
    │
    ├── 定义
    │   └── 一种运行在客户端（浏览器）的编程语言
    │
    └── 主要目的
        └── 实现人机交互效果
```

#### 核心作用

| 序号 | 作用 | 说明 |
|------|------|------|
| 1 | **网页特效** | 监听用户行为，让网页作出对应的反馈（例如动画、弹窗等） |
| 2 | **表单验证** | 针对表单数据的合法性进行判断（例如检查输入格式、必填项等） |
| 3 | **数据交互** | 获取后台（服务器）的数据，并将其渲染到前端页面 |
| 4 | **服务端编程** | 可以使用 Node.js 进行服务器端的开发 |

> 💡 **权威网站：** https://developer.mozilla.org/zh-CN/docs/Web/JavaScript

---

### 1.2 JavaScript 的组成

```
JavaScript
    │
    ├── ECMAScript
    │   └── JavaScript 语言基础
    │       ├── 变量
    │       ├── 分支语句
    │       ├── 循环语句
    │       ├── 对象
    │       └── 等等...
    │
    └── Web APIs
        ├── DOM (Document Object Model)
        │   └── 页面文档对象模型
        │       └── 操作文档，比如对页面元素进行移动、大小、添加删除等操作
        │
        └── BOM (Browser Object Model)
            └── 浏览器对象模型
                └── 操作浏览器，比如页面弹窗、检测窗口宽度、存储数据到浏览器等等
```

#### 组成部分详解

| 组成部分 | 定义 | 功能 |
|---------|------|------|
| **ECMAScript** | 规定了 JS 基础语法核心知识 | 提供变量、分支语句、循环语句、对象等基础语法 |
| **DOM** | 文档对象模型 | 操作文档，对页面元素进行移动、大小、添加删除等操作 |
| **BOM** | 浏览器对象模型 | 操作浏览器，如页面弹窗、检测窗口宽度、存储数据到浏览器等 |

### 1.4 JavaScript 书写位置

```
JavaScript 书写位置
    │
    ├── 1. 内部 JavaScript
    │   ├── 定义：直接写在 HTML 文件里，用 script 标签包住
    │   ├── 规范：script 标签写在 </body> 上面
    │   └── 示例：alert('你好,js') 页面弹出警告对话框
    │
    ├── 2. 外部 JavaScript
    │   ├── 定义：代码写在以 .js 结尾的文件里
    │   ├── 语法：通过 script 标签，引入到 HTML 页面中
    │   ├── 示例：<script src="my.js"></script>
    │   └── 优点：代码更加有序、更易于复用、HTML 更加易读
    │
    └── 3. 内联 JavaScript
        ├── 定义：代码写在标签内部
        ├── 示例：<button onclick="alert('逗你玩~~~')">点击我月薪过万</button>
        └── 注意：此处作为了解即可，但是后面 Vue 框架会用这种模式
```

#### 1.4.1 内部 JavaScript

**语法示例：**
```html
<body>
    <!-- 页面内容 -->
    <script>
        alert('嗨,欢迎来传智播学习前端技术!');
    </script>
</body>
```

**注意事项：**
- 浏览器会按照代码在文件中的顺序加载 HTML
- 如果先加载的 JavaScript 期望修改其下方的 HTML，可能会因为 HTML 尚未被加载而失效
- 因此，建议将 JavaScript 代码放在 HTML 页面底部附近（`</body>` 前面）是最佳策略

#### 1.4.2 外部 JavaScript

**语法示例：**
```html
<body>
    <!-- 通过 src 引入外部 js 文件 -->
    <script src="my.js"></script>
</body>
```

**注意事项：**
1. **script 标签中间无需写代码，否则会被忽略！**
2. 外部 JavaScript 会使代码更加有序、更易于复用，且没有了脚本的混合，HTML 也会更加易读，因此这是个好的习惯。

#### 1.4.3 内联 JavaScript

**语法示例：**
```html
<body>
    <button onclick="alert('逗你玩~~~')">点击我月薪过万</button>
</body>
```

**注意事项：**
- 此处作为了解即可，但是后面 Vue 框架会用这种模式

#### 📋 总结：JavaScript 三种书写位置

| 书写位置 | 定义 | 语法 | 适用场景 |
|---------|------|------|---------|
| **内部** | 直接写在 HTML 文件里 | `<script>代码</script>` | 小型项目、快速测试 |
| **外部** | 写在 .js 文件中 | `<script src="file.js"></script>` | 中大型项目（推荐） |
| **内联** | 写在标签内部 | `<button onclick="代码">` | 了解即可，Vue 框架会用到 |

#### ⚠️ 注意事项

1. **书写的位置尽量写到文档末尾 `</body>` 前面**
2. **外部 js 标签中间不要写代码，否则会被忽略**

### 1.5 JavaScript 注释

```
JavaScript 注释
    │
    ├── 单行注释
    │   ├── 符号：//
    │   ├── 作用：// 右边这一行的代码会被忽略
    │   ├── 快捷键：Ctrl + /
    │   └── 特点：一次只能注释一行，可以重复注释
    │
    └── 块注释（多行注释）
        ├── 符号：/* */
        ├── 作用：在 /* 和 */ 之间的所有内容都会被忽略
        ├── 快捷键：Shift + Alt + A
        └── 特点：可以注释多行内容
```

#### 1.5.1 单行注释

**语法：**
```javascript
<script>
// 这种是单行注释的语法
// 一次只能注释一行
// 可以重复注释
</script>
```

| 属性 | 说明 |
|------|------|
| **符号** | `//` |
| **作用** | `//` 右边这一行的代码会被忽略 |
| **快捷键** | `Ctrl + /` |

#### 1.5.2 块注释（多行注释）

**语法：**
```javascript
<script>
/* 这种的是多行注释的语法 */
/*
更常见的多行注释是这种写法
在些可以任意换行
多少行都可以
*/
</script>
```

| 属性 | 说明 |
|------|------|
| **符号** | `/* */` |
| **作用** | 在 `/*` 和 `*/` 之间的所有内容都会被忽略 |
| **快捷键** | `Shift + Alt + A` |

### 1.6 JavaScript 结束符

```
JavaScript 结束符
    │
    ├── 符号
    │   └── 使用英文的 ; 代表语句结束
    │
    ├── 实际情况
    │   └── 分号可写可不写，浏览器（JavaScript 引擎）可以自动推断语句的结束位置
    │
    ├── 现状
    │   └── 在实际开发中，越来越多的人主张，书写 JavaScript 代码时省略结束符
    │
    └── 约定
        └── 为了代码风格统一，团队内部需要达成一致，结束符要么每句都写，要么每句都不写（按照团队要求）
```

#### 代码示例对比

**带分号的写法：**
```javascript
<script>
alert(1);
alert(2);
</script>
```

**不带分号的写法：**
```javascript
<script>
alert(1)
alert(2)
</script>
```

> ⚠️ **注意：** 两种写法在 JavaScript 引擎的处理下通常是等效的，但为了代码风格统一，团队内部需要达成一致。

### 1.7 JavaScript 输入输出语法

**学习目标：** 能写出常见 JavaScript 输入输出语法

**语法定义：** 人和计算机打交道的规则约定，我们要按照这个规则去写。

#### 1.7.1 输出语法

```
JavaScript 输出语法
    │
    ├── 1. document.write()
    │   ├── 语法：document.write('要出的内容')
    │   ├── 作用：向 body 内输出内容
    │   └── 注意：如果输出的内容写的是标签，也会被解析成网页元素
    │
    ├── 2. alert()
    │   ├── 语法：alert('要出的内容')
    │   └── 作用：页面弹出警告对话框
    │
    └── 3. console.log()
        ├── 语法：console.log('控制台打印')
        └── 作用：控制台输出语法，程序员调试使用
```

##### 输出语法详解

| 语法 | 作用 | 使用场景 |
|------|------|---------|
| `document.write('要出的内容')` | 向 body 内输出内容 | 如果输出的内容写的是标签，也会被解析成网页元素 |
| `alert('要出的内容')` | 页面弹出警告对话框 | 提示用户信息 |
| `console.log('控制台打印')` | 控制台输出语法 | 程序员调试使用 |

**代码示例：**
```javascript
<script>
// 向 body 内输出内容
document.write('Hello World');
document.write('<h1>标题</h1>'); // 标签会被解析

// 页面弹出警告对话框
alert('这是一个警告对话框');

// 控制台输出（按 F12 打开开发者工具查看）
console.log('这是控制台输出');
</script>
```

#### 1.7.2 输入语法

```
JavaScript 输入语法
    │
    └── prompt()
        ├── 语法：prompt('请输入您的姓名:')
        └── 作用：显示一个对话框，对话框中包含一条文字信息，用来提示用户输入文字
```

**代码示例：**
```javascript
<script>

</script>
```

**展示效果：** 会弹出一个对话框，标题是"此网页显示"，内容是"请输入您的姓名:"，下方有一个文本输入框，以及"确定"和"取消"两个按钮。

#### 1.7.3 JavaScript 代码执行顺序

```
JavaScript 代码执行顺序
    │
    ├── 1. 按 HTML 文档流顺序执行
    │   └── 按 HTML 文档流顺序执行 JavaScript 代码
    │
    └── 2. alert() 和 prompt() 的特殊行为
        └── alert() 和 prompt() 它们会跳过页面渲染先被执行
            └── 注意：目前作为了解，后期讲解详细执行过程
```

**执行顺序说明：**
1. **按 HTML 文档流顺序执行：** JavaScript 代码会按照在 HTML 文档中出现的顺序依次执行
2. **alert() 和 prompt() 的特殊行为：** 这两个函数会跳过页面渲染先被执行（目前作为了解，后期讲解详细执行过程）

### 💻 二、变量 (Variables)

**学习目标：** 理解变量是计算机存储数据的"容器"

### 2.1 变量是什么？

```
变量
    │
    ├── 白话理解
    │   └── 变量就是一个装东西的盒子
    │
    ├── 通俗理解
    │   └── 变量是计算机中用来存储数据的"容器"，它可以让计算机变得有记忆
    │
    └── 注意事项
        └── 变量不是数据本身，它们仅仅是一个用于存储数值的容器。可以理解为是一个个用来装东西的纸箱子
```

**变量理解：**
- **变量是怎么理解？** 计算机中用来存储数据的"容器"，简单理解是一个个的盒子
- **变量有什么作用呢？** 用来存放数据的。注意变量指的是容器而不是数据
- **生活中的变量例子：** HTML标签、教室、宿舍等

### 2.2 变量的基本使用

#### 2.2.1 声明变量

```
声明变量
    │
    ├── 定义
    │   └── 要想使用变量，首先需要创建变量（也称为声明变量或者定义变量）
    │
    ├── 语法
    │   └── let 变量名
    │
    ├── 声明变量的构成
    │   ├── 声明关键字
    │   └── 变量名（标识）
    │
    └── 关键字 let 的解释
        └── let 即关键字（let:允许、许可、让、要），所谓关键字是系统提供的专门用来声明（定义）变量的词语
```

**代码示例：**
```javascript
<script>
// 声明一个 age 变量
let age
</script>
```

#### 2.2.2 变量赋值

```
变量赋值
    │
    ├── 定义
    │   └── 定义了一个变量后，你就能够初始化它（赋值）。在变量名之后跟上一个"="，然后是数值
    │
    ├── 语法
    │   └── 变量名 = 数值
    │
    └── 注意事项
        └── 注意：是通过变量名来获得变量里面的数据
```

**代码示例：**
```javascript
<script>
// 1. 声明一个 age 变量
let age
// 2. age 变量赋值为 18
age = 18
// 3. 输出 age 变量
alert(age)
</script>
```

**关键概念：**
- **变量：** `age`（在 `let age` 和 `age = 18` 中）
- **字面量：** `18`（在 `age = 18` 中）

#### 2.2.3 更新变量

```
更新变量
    │
    ├── 定义
    │   └── 变量赋值后，还可以通过简单地给它一个不同的值来更新它
    │
    └── let 关键字的特性
        └── let 不允许多次声明一个变量
```

**代码示例：**
```javascript
<script>
// 声明了一个 age 变量，同时里面存放了 18 这个数据
let age = 18
// 变量里面的数据发生变化更改为 19
age = 19
// 页面输出的结果为 19
document.write(age)
</script>
```

**错误示例（let 不允许多次声明）：**
```javascript
<script>
// 声明了一个 age 变量
let age = 18
// 这里不允许多次声明
let age = 19  // ❌ 错误：Uncaught SyntaxError: Identifier 'age' has already been declared
// 输出会报错
document.write(age)
</script>
```

#### 2.2.4 声明多个变量

```
声明多个变量
    │
    ├── 语法
    │   └── 多个变量中间用逗号隔开
    │
    ├── 示例（不推荐）
    │   └── let age = 18, uname = 'pink'
    │
    └── 说明与推荐
        ├── 说明：看上去代码长度更短，但并不推荐这样
        └── 推荐：为了更好的可读性，请一行只声明一个变量
```

**不推荐的写法：**
```javascript
<script>
let age = 18, uname = 'pink'
</script>
```

**推荐的写法：**
```javascript
<script>
```

### 5.1 为什么需要类型转换？

**学习目标：** 理解为什么需要进行类型转换

```
为什么需要类型转换
    │
    ├── JavaScript 的弱数据类型特性
    │   └── JavaScript 是弱数据类型：JavaScript 也不知道变量到底属于那种数据类型，只有赋值了才清楚
    │
    ├── 常见问题（坑）
    │   └── 使用表单、prompt 获取过来的数据默认是字符串类型的，此时就不能直接简单的进行加法运算
    │
    └── 通俗来说
        └── 就是把一种数据类型的变量转换成我们需要的数据类型
```

**代码示例：**
```javascript
<script>
// 字符串拼接，不是数字相加
console.log('10000' + '2000')  // 输出结果 100002000（字符串拼接）
// 此时需要转换变量的数据类型
</script>
```

**说明：**
- JavaScript 是弱数据类型：JavaScript 也不知道变量到底属于那种数据类型，只有赋值了才清楚
- 使用表单、prompt 获取过来的数据默认是字符串类型的，此时就不能直接简单的进行加法运算
- 此时需要转换变量的数据类型
- **通俗来说：** 就是把一种数据类型的变量转换成我们需要的数据类型

### 5.2 隐式转换

**学习目标：** 理解隐式转换的规则和特点

```
隐式转换
    │
    ├── 定义
    │   └── 某些运算符被执行时，系统内部自动将数据类型进行转换，这种转换称为隐式转换
    │
    ├── 规则
    │   ├── +号两边只要有一个是字符串，都会把另外一个转成字符串
    │   └── 除了+以外的算术运算符比如 - * / 等都会把数据转成数字类型
    │
    ├── 缺点
    │   └── 转换类型不明确，靠经验才能总结
    │
    └── 小技巧
        ├── +号作为正号解析可以转换成数字型
        └── 任何数据和字符串相加结果都是字符串
```

**代码示例：**
```javascript
<script>
// 数字相加
console.log(11 + 11)        // 22

// 字符串拼接（+号两边只要有一个是字符串，都会把另外一个转成字符串）
console.log('11' + 11)      // 1111

// 数字相减
console.log(11 - 11)        // 0

// 字符串转数字后相减（除了+以外的算术运算符都会把数据转成数字类型）
console.log('11' - 11)      // 0

// 数字相乘
console.log(1 * 1)          // 1

// 字符串转数字后相乘
console.log('1' * 1)        // 1

// 检测数据类型
console.log(typeof '123')   // string

// +号作为正号解析可以转换成数字型
console.log(typeof +'123')  // number
console.log(+'11' + 11)     // 22
</script>
```

**规则总结：**
1. **+号规则：** +号两边只要有一个是字符串，都会把另外一个转成字符串
2. **其他算术运算符：** 除了+以外的算术运算符比如 `- * /` 等都会把数据转成数字类型
3. **+号作为正号：** +号作为正号解析可以转换成数字型
4. **任何数据和字符串相加：** 任何数据和字符串相加结果都是字符串

**缺点：** 转换类型不明确，靠经验才能总结

#### 1.1.1 算术运算符优先级

**学习目标：** 能说出 JavaScript 算术运算符执行的优先级顺序

```
算术运算符优先级
    │
    ├── 定义
    │   └── 同时使用多个运算符编写程序时，会按着某种顺序先后执行，我们称为优先级
    │
    ├── 执行规则
    │   ├── JavaScript 中优先级越高越先被执行
    │   └── 优先级相同时以书从左向右执行
    │
    ├── 优先级顺序
    │   ├── 乘（*）、除（/）、取余（%）优先级相同
    │   ├── 加（+）、减（-）优先级相同
    │   └── 乘、除、取余优先级大于加、减
    │
    ├── 提升优先级
    │   └── 使用 () 可以提升优先级
    │
    └── 总结
        └── 先乘除后加减，有括号先算括号里面的
```

**优先级详情：**

| 运算符 | 优先级 | 结合性 | 说明 |
|--------|--------|--------|------|
| `()` | 最高 | - | 提高运算优先级，先计算括号内部 |
| `*`, `/`, `%` | 高 | 从左到右 | 乘法、除法、取余，优先级相同，从左到右计算 |
| `+`, `-` | 低 | 从左到右 | 加法、减法，优先级相同，从左到右计算 |

**代码示例：**
```javascript
<script>
let result1 = 2 + 3 * 4        // 结果为 14 (先乘后加: 3*4=12, 2+12=14)
let result2 = (2 + 3) * 4      // 结果为 20 (先括号内，后乘: 2+3=5, 5*4=20)
let result3 = 10 / 2 + 3 % 2   // 结果为 6 (先除和取余，再加: 10/2=5, 3%2=1, 5+1=6)
</script>
```

#### 1.3 一元运算符 (Unary Operators)
```
一元运算符
    │
    ├── 定义
    │   └── 只需要一个表达式就可以运算的运算符叫一元运算符
    │
    ├── 分类
    │   ├── 一元运算符（只需要一个表达式）

```

**使用场景：** 经常用于计数来使用。比如进行10次操作，用它来计算进行了多少次了

##### 1.3.1 自增运算符的用法

**前置自增：** `++变量名`
- 每执行1次，当前变量数值加1
- 其作用相当于 `num += 1`
- **记忆口诀：** ++在前先加（先自加再使用）

**后置自增：** `变量名++`
- 每执行1次，当前变量数值加1
- 其作用相当于 `num += 1`
- **记忆口诀：** ++在后后加（先使用再自加）

**代码示例：**
```javascript
<script>
let num = 1

// 前置自增
++num   // 让num的值加1变2
console.log(num)  // 2

// 后置自增
let num2 = 1
num2++   // 让num2的值加1变2
console.log(num2)  // 2

// 前置自增和后置自增单独使用没有区别
</script>
```

**重要说明：**
- 前置自增和后置自增独立使用时二者并没有差别！
- 一般开发中我们都是独立使用
- 后面 `i++` 后置自增会使用相对较多，并且都是单独使用

**前置自增和后置自增如果参与运算就有区别（难点，但是了解即可）：**

**前置自增：** 先自加再使用（记忆口诀：++在前先加）

```javascript
<script>
let i = 1
console.log(++i + 2)  // 结果是 4
// 注意：i是2
// i先自加1，变成2之后，再和后面的2相加
</script>
```

**后置自增：** 先使用再自加（记忆口诀：++在后后加）

```javascript
<script>
let i = 1
console.log(i++ + 2)  // 结果是 3
// 注意：此时的i是1
// 先和2相加，先运算输出完毕后，i再自加是2
</script>
```

##### 1.3.2 自减运算符的用法

**前置自减：** `--变量名`
- 每执行1次，当前变量数值减1
- 其作用相当于 `num -= 1`

**后置自减：** `变量名--`
- 每执行1次，当前变量数值减1
- 其作用相当于 `num -= 1`

**代码示例：**
```javascript
<script>
let num = 5

// 前置自减
--num   // 让num的值减1变4
console.log(num)  // 4

// 后置自减

num2--   // 让num2的值减1变4
console.log(num2)  // 4
</script>
```

#### 1.4 比较运算符 (Comparison Operators)

**学习目标：** 能够使用比较运算符比较两个数据

```
比较运算符
    │
    ├── 使用场景
    │   └── 比较两个数据大小、是否相等
    │
    └── 比较运算符列表
        ├── > 左边是否大于右边
        ├── < 左边是否小于右边
        ├── >= 左边是否大于或等于右边
        ├── <= 左边是否小于或等于右边
        ├── == 左右两边值是否相等
        ├── === 左右两边是否类型和值都相等
        └── !== 左右两边是否不全等
```

**说明：** 比较结果为 boolean 类型，即只会得到 `true` 或 `false`

**对比：**
- `=` 单等是赋值
- `==` 是判断，只要求值相等，不要求数据类型一样即可返回 true
- `===` 是全等，要求值和数据类型都一样返回的才是 true
- **开发中判断是否相等，强烈推荐使用 `===`**

**代码示例：**
```javascript
<script>
console.log(3 > 5)    // false
console.log(3 >= 3)   // true
console.log(2 == 2)   // true
console.log(2 == '2')  // true（值相等，类型不同也返回true）
console.log(2 === '2') // false（值和类型都要相等）
console.log(2 !== '2') // true（不全等）
</script>
```

**比较运算符的规则：**

1. **字符串比较，是比较的字符对应的ASCII码**
   - 从左往右依次比较
   - 如果第一位一样再比较第二位，以此类推
   - 比较的少，了解即可

2. **NaN不等于任何值，包括它本身**
   - 涉及到"NaN" 都是 false
   - `NaN === NaN` 返回 `false`

3. **尽量不要比较小数，因为小数有精度问题**

4. **不同类型之间比较会发生隐式转换**
   - 最终把数据隐式转换转成 number 类型再比较
   - 所以开发中，如果进行准确的比较我们更喜欢 `===` 或者 `!==`

#### 1.5 逻辑运算符 (Logical Operators)

#### 1.6 运算符优先级

**学习目标：** 掌握运算符优先级，能判断运算符执行的顺序

```
运算符优先级
    │
    ├── 优先级 1：小括号 ()
    │   └── 优先级最高
    │
    ├── 优先级 2：一元运算符 ++ -- !
    │   └── 一元运算符里面的逻辑非优先级很高
    │
    ├── 优先级 3：算数运算符
    │   └── 先 */% 后 +-
    │
    ├── 优先级 4：关系运算符 > >= < <=
    │
    ├── 优先级 5：相等运算符 == != === !==
    │
    ├── 优先级 6：逻辑运算符
    │   └── 先 && 后 ||
    │   └── 逻辑与比逻辑或优先级高
    │
    ├── 优先级 7：赋值运算符 =
    │
    └── 优先级 8：逗号运算符 ,
```

**优先级详情表：**

| 优先级 | 运算符 | 顺序 |
|--------|--------|------|
| 1 | 小括号 | `()` |
| 2 | 一元运算符 | `++ -- !` |
| 3 | 算数运算符 | 先 `* / %` 后 `+ -` |
| 4 | 关系运算符 | `> >= < <=` |
| 5 | 相等运算符 | `== != === !==` |
| 6 | 逻辑运算符 | 先 `&&` 后 `||` |
| 7 | 赋值运算符 | `=` |
| 8 | 逗号运算符 | `,` |

**注意事项：**
- 一元运算符里面的逻辑非优先级很高
- 逻辑与比逻辑或优先级高

**代码示例：**
```javascript
<script>
// 小括号优先级最高
console.log((2 + 3) * 4)  // 20

// 一元运算符优先级高
let num = 1
console.log(++num * 2)   // 4

// 逻辑与比逻辑或优先级高
console.log(true || false && false)  // true（先算 && 再算 ||）
</script>
```

### 📋 第二模块总结

| 问题 | 答案 |
|------|------|
| **运算符有哪些类型？** | 算术运算符、赋值运算符、一元运算符、比较运算符、逻辑运算符 |
| **运算符优先级从高到低是什么？** | 小括号 > 一元运算符 > 算数运算符 > 关系运算符 > 相等运算符 > 逻辑运算符 > 赋值运算符 > 逗号运算符 |
| **语句有哪些类型？** | 待补充 |

| 问题 | 答案 |
|------|------|
| **= 赋值运算符执行过程？** | 将等号右边的值赋予给左边，要求左边必须是一个容器 |
| **+= 出现是为了简化代码，比如让 `let num = 10`，`num` 加5怎么写呢？** | `num += 5` |
| **只需要一个表达式就可以运算的运算符叫什么？** | 一元运算符 |
| **自增运算符也是为了简化写法，每次自加1，使用场景是什么？** | 经常用于计数来使用，用来计算多少次 |
| **实际开发中，我们一般都是单独使用的，后置++更多** | 是的，后置自增 `i++` 使用相对较多，并且都是单独使用 |
| **= 和 == 和 === 怎么区别？** | `=` 是赋值；`==` 是判断，只要求值相等，不要求数据类型一样即可返回true；`===` 是全等，要求值和数据类型都一样返回的才是true。开发中，请使用 `===` |
| **比较运算符返回的结果是什么？** | 结果只有2个，`true` 或者 `false` |
| **逻辑与 (&&) 的特点和口诀？** | 符号两边都为true 结果才为true，口诀：一假则假 |
| **逻辑或 (\|\|) 的特点和口诀？** | 符号两边有一个true就为true，口诀：一真则真 |
| **逻辑非 (!) 的特点和口诀？** | true变false，false变true，口诀：真变假，假变真 |
| **运算符优先级从高到低是什么？** | 小括号 > 一元运算符 > 算数运算符 > 关系运算符 > 相等运算符 > 逻辑运算符 > 赋值运算符 > 逗号运算符 |
