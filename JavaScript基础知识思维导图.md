# 📜 JavaScript 基础知识思维导图

## 📌 1. JavaScript 是什么？

```
JavaScript
    │
    ├── 定义
    │   └── 一种运行在客户端（浏览器）的编程语言
    │
    └── 主要目的
        └── 实现人机交互效果
```

### 核心作用

| 序号 | 作用 | 说明 |
|------|------|------|
| 1 | **网页特效** | 监听用户行为，让网页作出对应的反馈（例如动画、弹窗等） |
| 2 | **表单验证** | 针对表单数据的合法性进行判断（例如检查输入格式、必填项等） |
| 3 | **数据交互** | 获取后台（服务器）的数据，并将其渲染到前端页面 |
| 4 | **服务端编程** | 可以使用 Node.js 进行服务器端的开发 |

> 💡 **权威网站：** https://developer.mozilla.org/zh-CN/docs/Web/JavaScript

## 🏗️ 2. JavaScript 的组成

```
JavaScript
    │
    ├── ECMAScript
    │   └── JavaScript 语言基础
    │       ├── 变量
    │       ├── 分支语句
    │       ├── 循环语句
    │       ├── 对象
    │       └── 等等...
    │
    └── Web APIs
        ├── DOM (Document Object Model)
        │   └── 页面文档对象模型
        │       └── 操作文档，比如对页面元素进行移动、大小、添加删除等操作
        │
        └── BOM (Browser Object Model)
            └── 浏览器对象模型
                └── 操作浏览器，比如页面弹窗、检测窗口宽度、存储数据到浏览器等等
```

### 组成部分详解

| 组成部分 | 定义 | 功能 |
|---------|------|------|
| **ECMAScript** | 规定了 JS 基础语法核心知识 | 提供变量、分支语句、循环语句、对象等基础语法 |
| **DOM** | 文档对象模型 | 操作文档，对页面元素进行移动、大小、添加删除等操作 |
| **BOM** | 浏览器对象模型 | 操作浏览器，如页面弹窗、检测窗口宽度、存储数据到浏览器等 |

## 🌐 3. JavaScript 运行环境

### 3.1 Chrome 浏览器运行环境

```
Chrome 浏览器运行环境
    │
    ├── 核心组件
    │   └── V8 引擎（负责解析和执行 JavaScript 代码）
    │
    └── 浏览器提供的 API/对象
        ├── DOM (Document Object Model)
        │   └── 用于操作 HTML 和 XML 文档的 API
```

**执行流程：** 待执行的 JS 代码 → V8 引擎 → 与 DOM、BOM、XHR 和内置对象等进行交互

```
Node.js 运行环境
    │

        │   └── Node.js 生态系统提供的其他各种模块和 API

            └── JavaScript 语言本身提供的标准内置对象
```

**执行流程：** 待执行的 JS 代码 → V8 引擎 → 与 fs、path、其他 API 和内置对象等进行交互

> 💡 **注意：** Node.js 可以用来模拟接口

## 📝 4. JavaScript 书写位置

```
JavaScript 书写位置
    │
    ├── 1. 内部 JavaScript
    │   ├── 定义：直接写在 HTML 文件里，用 script 标签包住
    │   ├── 规范：script 标签写在 </body> 上面
    │   └── 示例：alert('你好,js') 页面弹出警告对话框
    │
    ├── 2. 外部 JavaScript
    │   ├── 定义：代码写在以 .js 结尾的文件里
    │   ├── 语法：通过 script 标签，引入到 HTML 页面中
    │   ├── 示例：<script src="my.js"></script>
    │   └── 优点：代码更加有序、更易于复用、HTML 更加易读
    │
    └── 3. 内联 JavaScript
        ├── 定义：代码写在标签内部
        ├── 示例：<button onclick="alert('逗你玩~~~')">点击我月薪过万</button>
        └── 注意：此处作为了解即可，但是后面 Vue 框架会用这种模式
```

### 4.1 内部 JavaScript

**语法示例：**
```html
<body>
    <!-- 页面内容 -->
    <script>
        alert('嗨,欢迎来传智播学习前端技术!');
    </script>
</body>
```

**注意事项：**
- 浏览器会按照代码在文件中的顺序加载 HTML
- 如果先加载的 JavaScript 期望修改其下方的 HTML，可能会因为 HTML 尚未被加载而失效
- 因此，建议将 JavaScript 代码放在 HTML 页面底部附近（`</body>` 前面）是最佳策略

### 4.2 外部 JavaScript

**语法示例：**
```html
<body>
    <!-- 通过 src 引入外部 js 文件 -->
    <script src="my.js"></script>
</body>
```

**注意事项：**
1. **script 标签中间无需写代码，否则会被忽略！**
2. 外部 JavaScript 会使代码更加有序、更易于复用，且没有了脚本的混合，HTML 也会更加易读，因此这是个好的习惯。

### 4.3 内联 JavaScript

**语法示例：**
```html
<body>
    <button onclick="alert('逗你玩~~~')">点击我月薪过万</button>
</body>
```

**注意事项：**
- 此处作为了解即可，但是后面 Vue 框架会用这种模式

### 📋 总结：JavaScript 三种书写位置

| 书写位置 | 定义 | 语法 | 适用场景 |
|---------|------|------|---------|
| **内部** | 直接写在 HTML 文件里 | `<script>代码</script>` | 小型项目、快速测试 |
| **外部** | 写在 .js 文件中 | `<script src="file.js"></script>` | 中大型项目（推荐） |
| **内联** | 写在标签内部 | `<button onclick="代码">` | 了解即可，Vue 框架会用到 |

### ⚠️ 注意事项

1. **书写的位置尽量写到文档末尾 `</body>` 前面**
2. **外部 js 标签中间不要写代码，否则会被忽略**

## 💬 5. JavaScript 注释

```
JavaScript 注释
    │
    ├── 单行注释
    │   ├── 符号：//
    │   ├── 作用：// 右边这一行的代码会被忽略
    │   ├── 快捷键：Ctrl + /
    │   └── 特点：一次只能注释一行，可以重复注释
    │
    └── 块注释（多行注释）
        ├── 符号：/* */
        ├── 作用：在 /* 和 */ 之间的所有内容都会被忽略
        ├── 快捷键：Shift + Alt + A
        └── 特点：可以注释多行内容
```

### 5.1 单行注释

**语法：**
```javascript
<script>
// 这种是单行注释的语法
// 一次只能注释一行
// 可以重复注释
</script>
```

| 属性 | 说明 |
|------|------|
| **符号** | `//` |
| **作用** | `//` 右边这一行的代码会被忽略 |
| **快捷键** | `Ctrl + /` |

### 5.2 块注释（多行注释）

**语法：**
```javascript
<script>
/* 这种的是多行注释的语法 */
/*
更常见的多行注释是这种写法
在些可以任意换行
多少行都可以
*/
</script>
```

| 属性 | 说明 |
|------|------|
| **符号** | `/* */` |
| **作用** | 在 `/*` 和 `*/` 之间的所有内容都会被忽略 |
| **快捷键** | `Shift + Alt + A` |

## 🔚 6. JavaScript 结束符

```
JavaScript 结束符
    │
    ├── 符号
    │   └── 使用英文的 ; 代表语句结束
    │
    ├── 实际情况
    │   └── 分号可写可不写，浏览器（JavaScript 引擎）可以自动推断语句的结束位置
    │
    ├── 现状
    │   └── 在实际开发中，越来越多的人主张，书写 JavaScript 代码时省略结束符
    │
    └── 约定
        └── 为了代码风格统一，团队内部需要达成一致，结束符要么每句都写，要么每句都不写（按照团队要求）
```

### 代码示例对比

**带分号的写法：**
```javascript
<script>
alert(1);
alert(2);
</script>
```

**不带分号的写法：**
```javascript
<script>
alert(1)
alert(2)
</script>
```

> ⚠️ **注意：** 两种写法在 JavaScript 引擎的处理下通常是等效的，但为了代码风格统一，团队内部需要达成一致。

## 📥📤 7. JavaScript 输入输出语法

**学习目标：** 能写出常见 JavaScript 输入输出语法

**语法定义：** 人和计算机打交道的规则约定，我们要按照这个规则去写。

### 7.1 输出语法

```
JavaScript 输出语法
    │
    ├── 1. document.write()
    │   ├── 语法：document.write('要出的内容')
    │   ├── 作用：向 body 内输出内容
    │   └── 注意：如果输出的内容写的是标签，也会被解析成网页元素
    │
    ├── 2. alert()
    │   ├── 语法：alert('要出的内容')
    │   └── 作用：页面弹出警告对话框
    │
    └── 3. console.log()
        ├── 语法：console.log('控制台打印')
        └── 作用：控制台输出语法，程序员调试使用
```

#### 输出语法详解

| 语法 | 作用 | 使用场景 |
|------|------|---------|
| `document.write('要出的内容')` | 向 body 内输出内容 | 如果输出的内容写的是标签，也会被解析成网页元素 |
| `alert('要出的内容')` | 页面弹出警告对话框 | 提示用户信息 |
| `console.log('控制台打印')` | 控制台输出语法 | 程序员调试使用 |

**代码示例：**
```javascript
<script>
// 向 body 内输出内容
document.write('Hello World');
document.write('<h1>标题</h1>'); // 标签会被解析

// 页面弹出警告对话框
alert('这是一个警告对话框');

// 控制台输出（按 F12 打开开发者工具查看）
console.log('这是控制台输出');
</script>
```

### 7.2 输入语法

```
JavaScript 输入语法
    │
    └── prompt()
        ├── 语法：prompt('请输入您的姓名:')
        └── 作用：显示一个对话框，对话框中包含一条文字信息，用来提示用户输入文字
```

**代码示例：**
```javascript
<script>
// 显示输入对话框
prompt('请输入您的姓名:');
</script>
```

**展示效果：** 会弹出一个对话框，标题是"此网页显示"，内容是"请输入您的姓名:"，下方有一个文本输入框，以及"确定"和"取消"两个按钮。

### 7.3 JavaScript 代码执行顺序

```
JavaScript 代码执行顺序
    │
    ├── 1. 按 HTML 文档流顺序执行
    │   └── 按 HTML 文档流顺序执行 JavaScript 代码
    │
    └── 2. alert() 和 prompt() 的特殊行为
        └── alert() 和 prompt() 它们会跳过页面渲染先被执行
            └── 注意：目前作为了解，后期讲解详细执行过程
```

**执行顺序说明：**
1. **按 HTML 文档流顺序执行：** JavaScript 代码会按照在 HTML 文档中出现的顺序依次执行
2. **alert() 和 prompt() 的特殊行为：** 这两个函数会跳过页面渲染先被执行（目前作为了解，后期讲解详细执行过程）

## 📝 8. 字面量

**学习目标：** 能说出什么是字面量

```
字面量 (Literal)
    │
    ├── 定义
    │   └── 在计算机科学中，字面量 (literal) 是在计算机中描述事/物
    │
    └── 示例
        ├── 数字字面量：1000
        ├── 字符串字面量：'黑马程序员'
        ├── 数组字面量：[]（接下来会学）
        └── 对象字面量：{}（接下来会学）
```

### 字面量示例

| 类型 | 示例 | 说明 |
|------|------|------|
| **数字字面量** | `1000` | 我们工资是: 1000，此时 1000 就是数字字面量 |
| **字符串字面量** | `'黑马程序员'` | 用引号包裹的文本 |
| **数组字面量** | `[]` | 接下来会学 |
| **对象字面量** | `{}` | 接下来会学 |

**代码示例：**
```javascript
<script>
// 数字字面量
let salary = 1000;  // 1000 就是数字字面量

// 字符串字面量
let name = '黑马程序员';  // '黑马程序员' 就是字符串字面量

// 数组字面量（接下来会学）
let arr = [];

// 对象字面量（接下来会学）
let obj = {};
</script>
```

## 📚 总结

### JavaScript 是什么？
- JavaScript 是一门编程语言，可以实现很多的网页交互效果
- JavaScript 是一种运行在客户端（浏览器）的编程语言

### JavaScript 组成是什么？
- **ECMAScript**（基础语法）
- **Web APIs**（DOM、BOM）

### JavaScript 书写位置？
1. **内部 JavaScript** - 直接写在 HTML 文件里，用 script 标签包住，写到 `</body>` 标签上方
2. **外部 JavaScript** - 代码写在以 .js 结尾的文件里，通过 script 标签引入，但是 `<script>` 标签不要写内容，否则会被忽略
3. **内联 JavaScript** - 代码写在标签内部

### JavaScript 的注释？
1. **单行注释** - `//`（快捷键：Ctrl + /）
2. **多行注释** - `/* */`（快捷键：Shift + Alt + A）

### JavaScript 的结束符？
- **分号 `;`** - 可以加也可以不加，可以按照团队约定
- 为了代码风格统一，团队内部需要达成一致，结束符要么每句都写，要么每句都不写

### JavaScript 输入输出语句？
- **输入：** `prompt()` - 显示一个对话框，提示用户输入文字
- **输出：**
  - `alert()` - 页面弹出警告对话框
  - `document.write()` - 向 body 内输出内容
  - `console.log()` - 控制台输出语法，程序员调试使用

### JavaScript 代码执行顺序？
- 按 HTML 文档流顺序执行 JavaScript 代码
- `alert()` 和 `prompt()` 它们会跳过页面渲染先被执行（目前作为了解）

### 字面量是什么？
- 在计算机科学中，字面量 (literal) 是在计算机中描述事/物
- 常见类型：数字字面量、字符串字面量、数组字面量、对象字面量
