# 🌐 HTML 基础知识思维导图

## 📌 什么是 HTML？

```
HTML (HyperText Markup Language)
        │
        ├── 超文本标记语言
        │
        └── 用来告知浏览器如何组织页面的标记语言
```

### 核心知识点

| 序号 | 要点 |
|------|------|
| 1 | 标记也称为**标签**（元素） |
| 2 | 大小写都可以，建议**小写** |
| 3 | HTML 由一系列的**标签**组成 |

> 💡 **简单说：学习 HTML 就是学习标签**

---

## 📄 HTML 文档结构

```
<!DOCTYPE html>                    ← 文档类型声明
<html lang="zh-CN">               ← 根元素
    ┌─────────────────────────┐
    │ <head>                  │   ← 头部元素（元数据）
    │     <meta charset="UTF-8">
    │     <meta name="viewport" content="width=device-width, initial-scale=1.0">
    │     <title>页面标题</title>
    │ </head>                 │
    ├─────────────────────────┤
    │ <body>                  │   ← 页面内容
    │     ...显示内容...       │
    │ </body>                 │
    └─────────────────────────┘
</html>
```

### 各元素详解

| 元素 | 作用 | 说明 |
|------|------|------|
| `<!DOCTYPE html>` | 文档类型 | 告诉浏览器使用 HTML5 规范 |
| `<html>` | 根元素 | 包裹页面所有内容，`lang` 指定语言 |
| `<head>` | 头部元素 | 存放元数据，供机器处理 |
| `<meta charset>` | 字符集 | UTF-8 支持大多数语言字符 |
| `<meta viewport>` | 视口 | 移动端页面适配 |
| `<title>` | 标题 | 显示在浏览器标签页 |
| `<body>` | 主体 | 页面所有可见内容 |

### lang 属性

- `en` → 英语
- `zh-CN` → 简体中文

## 📝 标题和段落

### 标题标签 h1 ~ h6

```html
<h1>这是h1标题</h1>    ← 最大
<h2>这是h2标题</h2>
<h3>这是h3标题</h3>
<h4>这是h4标题</h4>
<h5>这是h5标题</h5>
<h6>这是h6标题</h6>    ← 最小
```

**特点：**
- ✅ 文字**加粗**显示
- ✅ 每行只显示**一个**（独占一行）
- ⚠️ `<h1>` 每个页面**只使用一次**（唯一性）
- 📌 每页标题层级**不超过三个**

### 段落标签 p

```html
<p>这是一个段落</p>
```

**特点：**
- 每行只显示一个
- 文字会自动换行
- 段落间有间距

## 💪 强调与重要性标签

HTML 提供了相应的标签，使其具有加粗、倾斜、下划线等效果，来着重**强调**某些文字。

### 语义化标签 ✅ 推荐使用

| 标签 | 作用 | 示例 |
|------|------|------|
| `<strong>` | **加粗** | `<strong>重要文字</strong>` |
| `<em>` | *倾斜* | `<em>强调文字</em>` |
| `<ins>` | <ins>下划线</ins> | `<ins>插入文字</ins>` |
| `<del>` | ~~删除线~~ | `<del>删除文字</del>` |

### 无语义标签 ❌ 不推荐

| 标签 | 作用 |
|------|------|
| `<b>` | 加粗 |
| `<i>` | 倾斜 |
| `<u>` | 下划线 |
| `<s>` | 删除线 |

> 💡 **尽量使用带有语义的标签！**

## 🧱 块级元素和内联元素

在 HTML 中有两种元素类别：**块级元素**和**内联元素**。分类不同决定着展示形式不一样。

```
              HTML 标签分类
                    │
        ┌───────────┴───────────┐
        │                       │
   ┌────▼────┐            ┌─────▼─────┐
   │ 块级元素 │            │  内联元素  │
   │ (Block) │            │ (Inline)  │
   └────┬────┘            └─────┬─────┘
        │                       │
   独占一行                 一行可放多个
```

### 块级元素 (Block)

| 特点 | 说明 |
|------|------|
| 独占一行 | 每个块级元素都从新行开始 |
| 可嵌套 | 可以嵌套其他元素 |
| 常见标签 | `<p>`, `<h1>`~`<h6>`, `<div>` 等 |

### 内联元素 (Inline)

| 特点 | 说明 |
|------|------|
| 一行多个 | 可以一行放多个，通常与文本一起使用 |
| 嵌套限制 | **不能**嵌套块级元素，可以嵌套其他内联元素 |
| 常见标签 | `<strong>`, `<em>`, `<a>` 等 |

### ⚠️ 注意事项

1. 段落 `<p>` 标签里面**不要**放其他块级元素
2. 段落里面主要放文字相关，比如内联元素
3. 元素分类不是一直不可更改的，后面可以通过 CSS 的 `display` 改变显示模式

## 🖼️ HTML 中的图片

### img 标签

```html
<img src="图片地址" alt="备选文本">
```

**特点：** 单标签（空元素）

### 图片属性总结

| 属性 | 作用 | 说明 |
|------|------|------|
| `src` | 图像地址 | 指向要插入到页面的图像地址 |
| `alt` | 备选文本 | 图片无法显示时提示的文字 |
| `width` | 宽度 | 设置图片宽度（建议 CSS 修改） |
| `height` | 高度 | 设置图片高度（建议 CSS 修改） |
| `title` | 图像标题 | 鼠标悬停显示的文本 |

**示例：**
```html
<img src="pig.png" alt="猪爸爸" title="猪爸爸出丑啦">
```

### ⚠️ 属性注意事项

1. 属性采取**键值对**形式：`属性="值"`
2. 属性之间没有必要的先后顺序
3. 属性之间必须有**空格分隔**

---

## 📸 常见图片格式

| 格式 | 特点 | 适用场景 |
|------|------|----------|
| **JPEG/JPG** | 有损压缩，放大缩小会变得模糊或有锯齿 | 摄影照片、网页图片（非透明背景） |
| **PNG** | 无损压缩，支持透明度 | Logo、网页图形、需要透明度的图像 |
| **GIF** | 支持动画，最多 256 色（索引色）| 简单动画、表情包、低色彩图形 |
| **WebP** | Google 开发，支持有损/无损压缩、透明度和动画 | 网页优化（替代 JPEG/PNG/GIF） |
| **AVIF** | 基于 AV1 视频编码，支持高压缩率和 HDR，压缩效率优于 WebP | 未来网页优化，需高性能压缩（B站、京东等） |

### 📋 图片格式选择指南

```
选择图片格式
     │
     ├── 🌐 网页优化 → 优先 WebP/AVIF（兼容时），备选 JPEG/PNG
     │
     ├── 🔲 透明图像 → PNG（静态）或 WebP（动态）
     │
     ├── 🎬 动画 → WebP/GIF（简单动画）
     │
     └── 📦 其他格式 → SVG、HEIC 等
```

> 💡 **开发中设计师提供图片，我们直接使用即可。**

## 📂 路径

在 HTML 中，**路径**用于指定文件（如图像、样式表或其他网页）的位置。

> 💡 **白话：** 图片相对于 HTML 文件的位置，HTML 怎么找到图片

```
                路径类型
                   │
        ┌──────────┴──────────┐
        │                     │
   ┌────▼────┐          ┌─────▼─────┐
   │ 相对路径 │          │  绝对路径  │
   └────┬────┘          └─────┬─────┘
        │                     │
   相对于当前             从根目录开始
   文件位置的路径         的完整路径
```

### 1️⃣ 相对路径（重点掌握）

相对于**当前文件位置**的路径。

| 场景 | 写法 | 示例 |
|------|------|------|
| **同一目录** | 直接使用文件名 或 `./文件名` | `<img src="pig.jpg">` 或 `<img src="./pig.jpg">` |
| **下级子目录** | `目录名/文件名` | `<img src="img/pig.jpg">` |
| **上级目录** | `../文件名` | `<img src="../pig.jpg">` |

**符号说明：**
- `./` → 当前文件夹
- `../` → 返回上一级目录
- `../../` → 返回上两级目录

**目录结构示例：**
```
project/
├── index.html        ← 当前文件
├── logo.png          ← 同级：src="logo.png"
├── images/
│   └── pig.jpg       ← 下级：src="images/pig.jpg"
└── ../
    └── pic.jpg       ← 上级：src="../pic.jpg"
```

### 2️⃣ 绝对路径（了解即可）

从**根目录**开始的完整路径，包含完整的 URL 地址。

**从盘符开始：**
```html
<img src="E:\HTML5\代码\pic.jpg" alt="示例图片">
```

**完整网络地址：**
```html
<img src="https://www.example.com/images/pic.jpg" alt="示例图片">
```

### 📋 路径对比

| 类型 | 特点 | 使用场景 |
|------|------|----------|
| **相对路径** | 相对当前文件，简洁灵活 | ✅ 推荐，项目内部文件引用 |
| **绝对路径** | 完整路径，不受位置影响 | 外部资源、CDN 引用 |

> ⚠️ **注意：** 开发中主要使用**相对路径**，更便于项目迁移和维护。

## 🎬 视频和音频

### 视频标签 video

`<video>` 元素用于在网页中轻松嵌入视频。

**基本语法：**
```html
<video src="video.mp4" controls width="300"></video>
```

### 视频属性

| 属性 | 作用 | 说明 |
|------|------|------|
| `src` | 视频地址 | 指向要插入的视频文件 |
| `controls` | 播放控件 | 显示浏览器自带播放控件 |
| `width/height` | 宽度/高度 | 设置视频的尺寸 |
| `autoplay` | 自动播放 | 需配合 `muted` 使用 |
| `muted` | 静音 | 静音播放 |
| `loop` | 循环播放 | 视频循环播放 |
| `poster` | 预览图像 | 视频加载前显示的封面图 |

**完整示例：**
```html
<video src="./media/video.mp4" width="400" controls autoplay muted loop poster="./media/cover.jpg"></video>
```

> 💡 **提示：** 在 H5 中，如果属性的键和值相同，可以省略值。如 `controls="controls"` 可简写为 `controls`。

### 视频格式兼容性

| 浏览器 | MP4 | WebM | Ogg |
|--------|-----|------|-----|
| Internet Explorer | ✅ YES | ❌ NO | ❌ NO |
| Chrome | ✅ YES | ✅ YES | ✅ YES |
| Firefox | ✅ YES | ✅ YES | ✅ YES |
| Safari | ✅ YES | ❌ NO | ❌ NO |
| Opera | ✅ YES (25+) | ✅ YES | ✅ YES |

> 📌 **HTML5 视频主要支持三种格式：MP4、Ogg、WebM，主要用 MP4 格式。**

### 视频兼容性写法

使用 `<source>` 元素提供多种格式，浏览器会播放第一个支持的格式：

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <source src="video.ogg" type="video/ogg">
  <source src="video.webm" type="video/webm">
  <p>您的浏览器不支持 HTML5 Video 标签，请升级浏览器。</p>
</video>
```

**要点：**
1. 将 `src` 属性放在单独的 `<source>` 元素中
2. 浏览器会检查 `<source>` 元素，播放第一个匹配的媒体
3. `type` 属性帮助浏览器快速跳过不支持的格式

`<audio>` 元素用于在网页中嵌入音频。

**基本语法：**

### 音频属性

| `muted` | 静音 |
| `loop` | 循环播放 |

> ⚠️ **注意：** 音频标签浏览器**不让自动播放**，后续可以用 JS 实现。

### 音频格式兼容性

|--------|-----|-----|-----|

| Chrome 6+ | ✅ YES | ✅ YES | ✅ YES |

| Safari 5+ | ✅ YES | ✅ YES | ❌ NO |
| Opera 10+ | ✅ YES | ✅ YES | ✅ YES |

> 📌 **音频主要用 MP3 格式。**

<audio controls>

## 🔗 创建超链接

超链接使我们能够将我们的文档链接到任何其他文档（或其他资源），也可以链接到文档的指定部分。

### 基本语法

```html
<a href=""></a>
```

**完整示例：**
```html
<a href="https://www.deepseek.com/">DeepSeek官网</a>
```

### href 属性

`href` 属性（也称为超文本引用或目标，它将包含一个网址）来创建一个基本链接。

**说明：**
- `href` 是超链接的**必需属性**
- 用于指定链接的目标地址
- 可以是 URL、文件路径或锚点

### 链接内容

链接可以包含除了自身之外的其他元素，比如：
- ✅ 文字
- ✅ 标题
- ✅ 图片
- ✅ 视频
- ✅ 其他内联元素

**示例：**
```html
<!-- 文字链接 -->
<a href="https://example.com">点击这里</a>

<!-- 图片链接 -->
<a href="https://example.com">
  <img src="logo.png" alt="Logo">
</a>
```

---

### 超链接的其他属性

#### 1️⃣ title 属性

**作用：** 鼠标悬停的提示文字

**示例：**
```html
<a href="https://example.com" title="这是提示文字">链接文本</a>
```

#### 2️⃣ target 属性

**作用：** 打开页面的方式

| 值 | 说明 |
|------|------|
| `_self` | 当前窗口打开（默认） |
| `_blank` | 新窗口打开 |

**示例：**
```html
<!-- 当前窗口打开（默认） -->
<a href="https://example.com" target="_self">当前窗口</a>

<!-- 新窗口打开 -->
<a href="https://example.com" target="_blank">新窗口</a>
```

---

### 不同类型的超链接

#### 1️⃣ 空链接

**定义：** 在 HTML 中，空链接通常指的是没有实际指向目标的超链接，符号是 `#`。

**语法：**
```html
<a href="#">商品列表</a>
```

**用途：**
- 占位链接
- 暂时没有目标地址时使用
- 配合 JavaScript 使用

#### 2️⃣ 下载链接

**定义：** 如果是 exe 或者压缩包点击是下载。

**语法：**
```html
<a href="download.exe">下载window版本</a>
<a href="file.zip">下载压缩包</a>
```

**说明：**
- 浏览器会直接下载文件，而不是打开
- 适用于可执行文件、压缩包等

#### 3️⃣ 邮件链接

**定义：** 点击后打开邮件客户端发送邮件。

**语法：**
```html
<a href="mailto:pig@mozilla.org">给我发邮件</a>
```

**⚠️ 注意事项：**
- 某些简单场景或个人使用情况下使用
- 公司或者流量大的网站**慎用**，有垃圾邮件等风险
- 建议使用其他方案替代（如表单提交）

### 锚点链接

#### 定义和用途

**锚点链接**允许用户在同一个页面内跳转到指定位置。

**特点：**
- ✅ 非常适合**长页面导航**
- ✅ 提升用户体验
- ✅ 快速定位到页面特定内容

**应用场景：**
- 长文章目录导航
- 单页应用（SPA）内部导航
- 常见问题（FAQ）页面
- 产品详情页的章节跳转

#### 实现方法

锚点链接的实现需要两个步骤：

##### 步骤 1：定义锚点目标

使用 `id` 属性创建锚点目标。

**语法：**
```html
<h2 id="1">第一部分</h2>
<div id="section-2">第二部分内容</div>
```

**说明：**
- 在目标元素上添加 `id` 属性
- `id` 值必须唯一
- 可以是任何元素（标题、div、section 等）

##### 步骤 2：创建跳转链接

使用 `#` 标记锚点目标。

**语法：**
```html
<a href="#1">跳转到第一部分</a>
<a href="#section-2">跳转到第二部分</a>
```

**说明：**
- `href` 属性值为 `#` + 目标元素的 `id` 值
- 点击链接后页面会跳转到对应位置

**完整示例：**
```html
<!-- 目录导航 -->
<nav>
  <a href="#intro">剧情介绍</a>
  <a href="#cast">演职员表</a>
  <a href="#characters">角色介绍</a>
  <a href="#awards">获奖记录</a>
</nav>

<!-- 页面内容 -->
<section id="intro">
  <h2>剧情介绍</h2>
  <p>内容...</p>
</section>

<section id="cast">
  <h2>演职员表</h2>
  <p>内容...</p>
</section>

<section id="characters">
  <h2>角色介绍</h2>
  <p>内容...</p>
</section>

<section id="awards">
  <h2>获奖记录</h2>
  <p>内容...</p>
</section>
```

### 页面滑动效果（平滑滚动）

想要点击链接之后，页面具有滑动效果，可以在 `<head>` 标签中添加以下 CSS 代码：

**实现方法：**
```html
<head>
  <style>
    html {
      scroll-behavior: smooth;
    }
  </style>
</head>
```

**说明：**
- `scroll-behavior: smooth;` 实现平滑滚动效果
- 添加到 `html` 元素上
- 适用于所有锚点链接和页面内跳转

**效果：**
- 点击锚点链接时，页面会平滑滚动到目标位置
- 而不是瞬间跳转
- 提升用户体验

---

## 📋 超链接总结

| 类型 | 语法 | 用途 |
|------|------|------|
| **普通链接** | `<a href="URL">文本</a>` | 跳转到其他页面或网站 |
| **空链接** | `<a href="#">文本</a>` | 占位链接，配合 JS 使用 |
| **下载链接** | `<a href="file.exe">下载</a>` | 下载文件 |
| **邮件链接** | `<a href="mailto:email">发邮件</a>` | 打开邮件客户端 |
| **锚点链接** | `<a href="#id">跳转</a>` | 页面内跳转 |

**常用属性：**
- `href` - 链接地址（必需）
- `title` - 悬停提示文字
- `target` - 打开方式（`_self` / `_blank`）

**最佳实践：**
- ✅ 使用语义化的链接文本
- ✅ 外部链接使用 `target="_blank"` 在新窗口打开
- ✅ 长页面使用锚点链接提升导航体验
- ✅ 添加平滑滚动效果提升用户体验
- ⚠️ 邮件链接慎用，避免垃圾邮件风险

网页的外观多种多样，但是大概都包含：页眉、导航栏、主内容、侧边栏、页脚等。HTML 提供了相应的标签来组织这些结构。

        │                     │
    HTML5新增           布局时使用

### 🏛️ 网站结构标签

网站结构标签是 HTML5 新增的语义化标签，用于更好地组织和描述网页的各个区域。

#### 标签列表

| 标签 | 作用 | 说明 |
|------|------|------|
| `<header>` | 页眉（头部） | 用于表示网页的页眉，通常包含网站标题、Logo 等 |
| `<main>` | 主要内容 | 用于表示网页的主要内容，**每个页面只能有一个** |
| `<nav>` | 导航栏 | 用于表示导航栏，包含导航链接 |
| `<article>` | 独立文章 | 用于表示独立、完整的文章内容 |
| `<section>` | 文档分块 | 用于表示文档中的一个独立的分块或区域 |
| `<aside>` | 侧边栏 | 用于表示与主要内容相关但可以独立存在的内容，通常作为侧边栏 |
| `<footer>` | 页脚（底部） | 用于表示页面的页脚，通常包含版权信息、联系方式等 |

#### 布局示意图

```
┌─────────────────────────────────┐
│         <header>                │  ← 页眉
├─────────────────────────────────┤
│          <nav>                  │  ← 导航栏
├──────────────┬──────────────────┤
│              │                  │
│   <aside>    │    <main>        │  ← 侧边栏 + 主内容
│   (侧边栏)   │    <article>     │
│              │    <section>     │
│              │                  │
├──────────────┴──────────────────┤
│         <footer>                │  ← 页脚
└─────────────────────────────────┘
```

#### 完整示例

```html
<!DOCTYPE html>
<html lang="zh-CN">
<head>
  <meta charset="UTF-8">
  <title>网站结构示例</title>
</head>
<body>
  <header>
    <h1>网站标题</h1>
    <p>网站副标题</p>
  </header>
  
  <nav>
    <ul>
      <li><a href="#home">首页</a></li>
      <li><a href="#about">关于</a></li>
      <li><a href="#contact">联系</a></li>
    </ul>
  </nav>
  
  <main>
    <article>
      <h2>文章标题</h2>
      <p>文章内容...</p>
    </article>
    
    <section>
      <h2>相关章节</h2>
      <p>章节内容...</p>
    </section>
  </main>
  
  <aside>
    <h3>侧边栏</h3>
    <p>相关内容...</p>
  </aside>
  
  <footer>
    <p>© 2024 版权所有</p>
  </footer>
</body>
</html>
```

#### ⚠️ 兼容性说明

- 这些标签是 **HTML5 新增的标签**
- **兼容性：** 这些标签受浏览器兼容性问题影响
  - **PC 端：** 根据公司需求决定是否使用
  - **移动端：** 放心使用

> 💡 **建议：** 在移动端开发中优先使用这些语义化标签，提升代码可读性和 SEO 效果。

### 📦 无语义标签

没有合适语义标签时，在进行一些布局时候可以选择：`<div>` 和 `<span>` 标签。

#### 1️⃣ div 标签

**定义：** `<div>` 是一个块级无语义容器元素。

**特点：**
1. **块级元素：** 默认独占一行，前后会自动换行
2. **布局结构：** 通常用于布局结构，作为其他元素的容器
3. **内容包含：** 可以包含其他块级或行内元素
4. **无语义：** 默认没有语义

**语法：**
```html
<div>内容</div>
```

**示例：**
```html
<div class="container">
  <div class="header">页眉</div>
  <div class="content">内容</div>
  <div class="footer">页脚</div>
</div>
```

**用途：**
- 页面布局容器
- 分组相关元素
- 配合 CSS 进行样式控制

#### 2️⃣ span 标签

**定义：** `<span>` 是一个行内无语义容器元素。

**特点：**
1. **行内元素：** 不会换行，仅包裹内容的一部分
2. **局部样式/操作：** 用于对文本或行内元素的局部样式或操作
3. **无语义：** 默认没有语义

**语法：**
```html
<span>内容</span>
```

**示例：**
```html
<p>这是一段文字，其中<span class="highlight">这部分</span>需要高亮显示。</p>
```

**用途：**
- 文本局部样式控制
- 配合 JavaScript 进行操作
- 内联元素的容器

#### 📋 div 和 span 对比

| 特性 | `<div>` | `<span>` |
|------|---------|----------|
| **元素类型** | 块级元素 | 行内元素 |
| **换行** | 独占一行 | 不换行 |
| **主要用途** | 布局结构、容器 | 文本局部样式、操作 |
| **可包含** | 块级和行内元素 | 只能包含行内元素 |
| **语义** | 无语义 | 无语义 |

> 💡 **提示：** 虽然 `<div>` 和 `<span>` 无语义，但在实际开发中仍然非常重要，是布局的基础工具。

### 📋 列表标签

HTML 列表是网页内容组织的重要元素。可以让我们显示内容更加整齐有序。

#### 列表类型

```
              列表标签
                 │
     ┌───────────┼───────────┐
     │           │           │
┌────▼────┐ ┌────▼────┐ ┌────▼────┐
│ 无序列表 │ │ 有序列表 │ │ 描述列表 │
│   ul   │ │   ol   │ │   dl   │
└────────┘ └────────┘ └────────┘
```

**重要性排序：**
1. ⭐⭐⭐ **无序列表** - 重点记住
2. ⭐⭐ **描述列表** - 其次重要
3. ⭐ **有序列表** - 了解即可

---

#### 1️⃣ 无序列表 `<ul>`

**定义：** 无序列表用于表示顺序无关紧要的项目列表。

**特点：**
- ✅ **顺序无关：** 列表项的顺序不重要
- ✅ **默认标记：** 浏览器会为列表项添加实心圆点（`disc`）

**基本语法：**
```html
<ul>
  <li>列表项1</li>
  <li>列表项2</li>
  <li>列表项3</li>
</ul>
```

**示例：**
```html
<ul>
  <li>猪爸爸</li>
  <li>猪妈妈</li>
  <li>佩奇</li>
  <li>乔治</li>
</ul>
```

**渲染效果：**
- 猪爸爸
- 猪妈妈
- 佩奇
- 乔治

**组成元素：**

| 元素 | 作用 | 说明 |
|------|------|------|
| `<ul>` | 定义无序列表的**容器** | 只能包含 `<li>` 元素 |
| `<li>` | 定义列表中的**选项**（列表项） | 里面可以放置其他 HTML 元素 |

**⚠️ 注意事项：**
1. `<ul>` 里面**只能包含** `<li>` 元素
2. `<li>` 里面**可以放其他元素**（文本、图片、链接、段落等）
3. 无序列表在**开发布局中非常重要**

**嵌套示例：**
```html
<ul>
  <li>
    <h3>标题</h3>
    <p>描述文字</p>
    <a href="#">链接</a>
  </li>
  <li>
    <img src="image.jpg" alt="图片">
    <p>图片说明</p>
  </li>
</ul>
```

---

#### 2️⃣ 有序列表 `<ol>`

**定义：** 有序列表用于表示有先后顺序的项目列表。

**特点：**
- ✅ **顺序有关：** 列表项的顺序很重要，例如步骤、排名等
- ✅ **默认编号：** 浏览器会自动为列表项添加数字编号（1., 2., 3., ...）

**基本语法：**
```html
<ol>
  <li>列表项1</li>
  <li>列表项2</li>
  <li>列表项3</li>
</ol>
```

**示例：**
```html
<ol>
  <li>看视频</li>
  <li>写代码</li>
  <li>做笔记</li>
  <li>多复习</li>
</ol>
```

**渲染效果：**
1. 看视频
2. 写代码
3. 做笔记
4. 多复习

**组成元素：**

| 元素 | 作用 | 说明 |
|------|------|------|
| `<ol>` | 定义有序列表的**容器** | 只能包含 `<li>` 元素 |
| `<li>` | 定义列表中的**选项**（列表项） | 里面可以放置其他 HTML 元素 |

**常用属性（了解）：**

| 属性 | 作用 | 示例 | 说明 |
|------|------|------|------|
| `type` | 设置列表项的编号类型 | `type="a"` (小写字母), `type="A"` (大写字母), `type="i"` (小写罗马数字), `type="I"` (大写罗马数字), `type="1"` (数字，默认) | 更改列表标记样式 |
| `start` | 设置列表的起始值 | `start="5"` | 列表从指定数字开始编号 |
| `reversed` | 反转列表项的顺序 | `<ol reversed>` | 列表项倒序显示 |

**示例（带属性）：**
```html
<ol type="A" start="3">
  <li>任务 C</li>
  <li>任务 D</li>
  <li>任务 E</li>
</ol>
```

**渲染效果：**
C. 任务 C
D. 任务 D
E. 任务 E

---

#### 3️⃣ 描述列表 `<dl>`

**定义：** 描述列表用于标记一组项目及相关描述。

**特点：**
- ✅ **术语-描述关系：** 适用于词汇表、元数据或问答形式
- ✅ **一对多关系：** 一个术语可以有多个描述，一个描述也可以对应多个术语

**基本语法：**
```html
<dl>
  <dt>术语</dt>
  <dd>术语的描述</dd>
</dl>
```

**示例：**
```html
<dl>
  <dt>家电</dt>
  <dd>电视</dd>
  <dd>冰箱</dd>
  <dd>空调</dd>
  <dd>烟灶</dd>
</dl>
```

**渲染效果：**
家电
  电视
  冰箱
  空调
  烟灶

**组成元素：**

| 元素 | 作用 | 说明 |
|------|------|------|
| `<dl>` | 定义列表的**容器** | 包含 `dt` 和 `dd` 元素 |
| `<dt>` | 定义**术语**（definition term） | 通常是列表中的标题或概念，通常显示为左对齐或加粗 |
| `<dd>` | 定义**描述**（definition description） | 对前面 `dt` 元素的描述或解释，通常显示为缩进形式 |

**⚠️ 注意事项：**
1. `<dl>` 里面**只能包含** `<dt>` 和 `<dd>` 元素
2. **一个 `<dt>` 可以对应多个 `<dd>`**
3. `<dd>` 里面**可以放其他元素**（段落、图片、链接等）

**复杂示例：**
```html
<dl>
  <dt>咖啡</dt>
  <dd>一种提神饮品。</dd>
  <dd>通常是热的。</dd>
  
  <dt>牛奶</dt>
  <dd>富含钙质的白色液体。</dd>
  <dd>
    <p>可以用于制作各种饮品。</p>
    <img src="milk.jpg" alt="牛奶">
  </dd>
</dl>
```

**应用场景：**
- 词汇表
- 元数据（如作者、日期等）
- 常见问题（FAQ）
- 术语定义

---

## 📋 列表标签总结

| 标签 | 用途 | 构成 | 语义 | 默认样式 | 顺序 | 重要性 |
|------|------|------|------|----------|------|--------|
| `<ul>` | 无序列表 | `ul` + `li` | 强调项目组 | 实心圆点 | 无关 | ⭐⭐⭐ 重点 |
| `<ol>` | 有序列表 | `ol` + `li` | 强调顺序 | 数字编号 | 有关 | ⭐ 了解 |
| `<dl>` | 描述列表 | `dl` + `dt` + `dd` | 术语-描述 | 无特殊符号，通常缩进 | 无关 | ⭐⭐ 其次 |

**最佳实践：**
- ✅ 根据内容的**逻辑关系**选择合适的列表类型
- ✅ `<li>` 元素可以包含块级或内联元素
- ✅ 列表的样式（如编号类型、符号）通常使用 CSS 进行控制，而非 `type` 属性
- ✅ 无序列表在**开发布局中非常重要**，常用于导航菜单、功能列表等

---

## 📋 布局标签总结

| 类别 | 标签 | 特点 | 用途 |
|------|------|------|------|
| **网站结构** | `<header>`, `<nav>`, `<main>`, `<article>`, `<section>`, `<aside>`, `<footer>` | 语义化，HTML5 新增 | 组织页面结构 |
| **无语义容器** | `<div>`, `<span>` | 无语义，但重要 | 布局和样式控制 |
| **列表** | `<ul>`, `<ol>`, `<dl>` | 组织内容 | 显示整齐有序的内容 |

**选择建议：**
- ✅ 优先使用**语义化标签**（网站结构标签）
- ✅ 没有合适语义时使用 `<div>` 和 `<span>`
- ✅ 根据内容逻辑选择合适的列表类型
